你是“提示词系统架构师 + 影视脚本导演”。我要为一个本地流水线项目（Tieba Story Video）设计一套可扩展的提示词模板集，用来把单个贴吧故事工程化生成约 2 分钟短片。

你的输出必须是 JSON（只输出 JSON，不要解释），顶层字段：
- version (string)
- goals (array of string)
- global_rules (array of string)
- templates (object)，键为模板名，值为对象 {purpose, inputs, outputs, system_prompt, user_prompt_template}

约束与目标：
1) 风格与情感
   - 旁白要更有感染力、更贴近作者在发帖时的情绪起伏，但要避免过度煽情与狗血
   - 叙事结构明确，节奏有起伏，结尾要收束有力
2) 输出可执行
   - 所有模板都要求输出可落盘、可验证的结构化 JSON
   - 分镜必须可执行：每镜头都能生成 1 张静帧图 + 1 段旁白
3) 可追溯
   - 所有 LLM 输出都必须带 trace（与楼层/段落的映射）
4) 可扩展性（重点）
   - 模板必须支持参数化：target_total_seconds、scene_count、chars_per_second、style_preset、persona 等
   - 模板必须支持多版本与多风格：例如 documentary / diary / noir / warm 等
5) 本次要覆盖的模板（至少 6 个）
   - summarize_v3：从 floors.normalized + (可选) vlm.image_descriptions 生成 story.summary.json
   - storyboard_v3：从 story.summary.json 生成 storyboard.json（scene_count 固定，且总时长约 120s）
   - voiceover_polish_v1：对 storyboard.json 的 voiceover_text 做二次打磨（更有感染力，仍克制）
   - image_prompt_polish_v1：对 storyboard.json 中每个 scene 的 positive/negative 做统一风格与一致性增强
   - bgm_plan_v2：从 summary + storyboard 生成 bgm.plan.json（风格、BPM、混音建议、风险）
   - qa_validate_v1：对产物进行自检（字段/时长/可执行性/禁忌项）并输出 JSON 报告

具体输出字段要求（务必在 system_prompt 中写清楚）：
- story.summary.json: post_url, post_id, title, synopsis, paragraphs[], trace[]
- storyboard.json: post_url, post_id, title, synopsis, scenes[固定 N=10], 每条 scene_id, duration_s, visual_description, composition, mood, voiceover_text, prompt{positive,negative}, trace{paragraph_ids, floors{start,end,indexes?}}
- bgm.plan.json: mood, bpm_range, style_keywords[], mixing{ducking_db, target_lufs, notes}, candidates[], selected, risks[]
- qa report: pass, errors[], warnings[], stats{total_seconds, scenes_count, avg_chars_per_second}

禁忌：
- 不要输出任何真实姓名、具体地名/学校名、账号/昵称、可识别个人信息
- 不要输出任何 API Key、Token、Cookie
- 不要包含 Markdown 代码块标记

输入给你的是：项目目标与约束（无需额外输入）。
